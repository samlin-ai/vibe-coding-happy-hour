# 教練指南：用 Gemini CLI 帶孩子做出第一款 Vibe Coding 遊戲

[English](./coach-guide-gemini-cli.md) | 繁體中文

**對象：** 孩子（8–14 歲）＋分組教練 | **團隊人數：** 3–5 個孩子 | **時長：** 60 分鐘

---

## 關鍵成果

教練不只引導團隊合作交付這些關鍵成果，也要挑戰他們超越目標。

1. [ ] 為團隊取名。
2. [ ] 腦力激盪遊戲點子與提示詞。
3. [ ] 腦力激盪改進方向與提示詞。
4. [ ] 理解遊戲如何運作及其設計。
5. [ ] 排練成果發表（Show & Tell）。

---

## 流程時間表

**教練秘訣：** 第四段「打造個人特色」是加分項。依團隊進度自由增減時間。讓團隊做出遊戲並準備好成果發表更重要。

### 第一段 — 建造者集合（0:00–0:05）

**目標：** 讓孩子們互相認識，一起為團隊取名。

---

### 第二段 — 點燃靈感（0:05–0:15）

**目標：** 讓孩子們興奮起來，一起選出隊名與遊戲概念。

**對團隊說：**
> 「今天你們都是遊戲設計師。一小時內，你們會做出一款任何人都能玩的真正遊戲。AI 負責寫程式——你們的工作是告訴它要做『什麼』。」

**活動 — 3 分鐘獨立思考＋集體腦力激盪：**

1. 請每個孩子先自己思考一分鐘，把答案寫在便利貼上：*「你最愛的遊戲是什麼？」*
2. 每個孩子分享答案，再以團隊為單位分類整理。
3. 舉手投票選出前兩名。
4. 全組一起把前兩名結合成一款新遊戲。

**教練秘訣：** 結合獨立思考與集體腦力激盪（「混合式」做法）能激發更好的團隊點子，也能避免群體迷思。

---

### 第三段 — 打造第一版（0:15–0:35）

**目標：** 20 分鐘內讓螢幕上出現一款能玩的遊戲。

**步驟 1 — 生成遊戲（孩子們輪流下提示詞）：**

例如：
```
不使用任何框架，做一個簡單的俄羅斯方塊網頁遊戲。
```

**步驟 2 — 在瀏覽器打開，讓每個孩子都玩玩看**

**教練秘訣：** 如果遊戲跑不起來，別慌！說：*「連專業開發者都會遇到 bug。我們請 Gemini CLI 修好它——這就是 vibe coding 的超能力。」*

**快速修復提示詞範本：**
```
遊戲有一個 bug：[用白話描述哪裡不對]。
請修好它。
```

**步驟 3 — 提交（Commit）變更**

Git 是程式碼的時光機。它記錄每一次提交的變更，讓你和 AI 都擁有完整的歷程紀錄。

提交每次變更的方式：[Source Control] > 用提示詞當作訊息（Message） > [Commit]。

**教練秘訣：** 有時間的話，打開[使用版本控制](https://docs.cloud.google.com/shell/docs/version-control)快速介紹一下。

---

### 第四段 — 打造個人特色（0:35–0:45）

**目標：** 每個孩子加上一個個人巧思。這是創意大爆發的時刻。

**輪流進行。每個孩子選「一個」升級：**

孩子們可以自由混搭的提示詞範例：
```
玩家得分時加一個有趣的音效。
```
```
加一個從 30 秒開始倒數的計時器。
```
```
把玩家角色換成表情符號，例如 🐸，不要用方塊。
```
```
加一個會在每局之間保存的最高分。
```
```
每得 10 分讓遊戲變快一點。
```
```
把背景色改成 [顏色]，字體換成有趣一點的。
```
```
在遊戲結束畫面加一個「再玩一次」按鈕。
```

**教練秘訣：** 讓提示詞寫手（Prompt Writer）在輸入前先把每個升級需求唸給全組聽。這能讓大家保持參與，也讓孩子們在出錯前就發現誤解。

---

### 第五段 — 理解遊戲（0:45–0:53）

**目標：** 理解遊戲如何運作、如何構建。

引導孩子們先嘗試自己回答這些問題，再問 Gemini。

1. 請團隊先思考並猜猜遊戲怎麼運作。
2. 打開 `index.html` 和其他生成的檔案快速看一看。
3. 請 Gemini 用 [x] 歲小朋友能懂的方式解釋遊戲怎麼運作。
4. 請 Gemini 畫一張遊戲運作的 ASCII 系統圖，再追問後續問題。

---

### 第六段 — 排練成果發表（0:53–1:00）

**目標：** 慶祝他們的成果並反思。

**展示遊戲：**
- 讓每個孩子玩 30 秒。
- 記錄員（Note Taker）唸出他們新增的所有功能。
- 拍一張團體照或截圖。

**回顧提問（挑 2–3 個）：**
1. *「最難的部分是什麼？你們怎麼解決的？」*
2. *「如果再多一小時，你會加什麼？」*
3. *「程式是誰寫的——你還是 AI？誰是主導？」*（是他們！）
4. *「還有哪裡可以用上這樣的 AI 助手？」*

**頒發勝利：**
> 「你們剛剛做出了一款真正的遊戲。你們正式成為遊戲製作人了。」

## 常見狀況處理

### 「Gemini 給了奇怪的程式碼／跑不起來」
平常心面對：bug 很正常。用這個提示詞：
```
結果和預期不一樣。[描述問題]。
可以試試更簡單的做法嗎？
```

### 「孩子們在吵要加什麼功能」
用投票規則：每個孩子一票，多數決。平手由遊戲設計師（Game Designer）角色決定。

### 「有孩子說『我做不到，我不會寫程式』」
回答：*「這就是重點——你不需要會！你只要用白話告訴 AI 你想要什麼。你剛剛已經做到了。」*

### 「時間不夠了」
跳過第四段的升級，0:45 直接進成果發表。能玩的遊戲贏過沒做完的華麗遊戲。

### 「時間多出來了」
問：*「要不要加雙人模式？我們問問 Gemini！」*

## 快速參考：好用的 Gemini CLI 提示詞

```
# 從零開始做新遊戲
我們是正在做 [類型] 遊戲的小朋友。請只用 HTML/CSS/JS 建立 index.html。

# 修 bug
有一個 bug：[描述它]。請修好 index.html 並顯示完整更新後的檔案。

# 加功能
幫我們的遊戲加上 [功能]。顯示更新後的 index.html。

# 解釋東西
用 10 歲小朋友能懂的簡單話解釋：[程式碼片段] 在做什麼？

# 要點子
我們的 [類型] 遊戲可以加哪 3 個有趣的東西？保持簡單。

# 變漂亮
讓遊戲看起來更繽紛、更刺激。用明亮的顏色和表情符號。
```

## 成功的樣子

一小時結束時，團隊應該有：
- 一款在瀏覽器中能玩的遊戲。
- 每個孩子至少貢獻了一個功能。
- 一次與 AI 迭代的經驗（提示 → 測試 → 改進）。
- 「我能用程式做出東西」的自信。

遊戲不必完美。**完成勝過完美。**

## 加分題

團隊有多餘時間時可以試試。

### 問題 1：如何施展更強的咒語（也就是提示工程）？

**「角色、目標、情境」**提示框架
> 「你是一位專業遊戲開發者。請做一個簡單的網頁遊戲：熊貓接住掉落的墨西哥捲餅。要色彩繽紛、有趣，並且能在瀏覽器執行。」

追問後續問題，例如：
> 「為什麼這個框架效果更好？」

提示：這和**六頂思考帽的角色扮演**有什麼相似之處？因為 **Attention Is All You Need**。

### 問題 2：如何用 AI 之道還治 AI 之身？

**元提示（Meta Prompting）**——直接請它幫你升級提示詞：
> 「你是最強的提示工程師。請寫一個提示詞，用來建立一個簡單的網頁遊戲：熊貓接住掉落的墨西哥捲餅。遊戲要色彩繽紛、有趣，並且能在瀏覽器執行。」
